# Security Policy

## 1. Purpose
Define the security controls, responsibilities, and expectations for AutoBuddy.

## 2. Scope
Applies to the AutoBuddy application, backend services, database systems, and any connected infrastructure.

## 3. Authentication and Authorization
- Use strong password policies.
- Require role-based access controls for admin/driver/passenger functions.
- Protect API endpoints with bearer tokens.

## 4. Data Protection
- Encrypt sensitive data in transit (HTTPS/TLS).
- Store credentials hashed and salted.
- Restrict access to personal data and KYC documents.

## 5. Incident Response
- Monitor logs for unusual activity.
- Escalate breaches immediately to security and operations.
- Preserve forensic data and remediate vulnerabilities.

## 6. Audit and Compliance
- Maintain audit logs for admin actions and payment events.
- Review permission assignments regularly.
- Align with applicable local privacy and transport regulations.
