Code + competitor reference

AutoBuddy

A full-stack ride operations platform built from the repo and aligned with the shared Uber/Ola/Rapido comparison reference.

Market problem

Uncertain local rides

Passengers face unclear fares, poor ETA accuracy, inconsistent communication, and safety concerns.

Bad driver economics

High commissions, manual payouts, poor earnings visibility, and low trust push drivers away.

Operators are fragmented

Local teams need pricing, KYC, compliance, analytics, and support workflows in one system.

AutoBuddy solution

Passenger, driver & admin together

Passenger and driver Expo apps plus FastAPI backend with admin and operations routers for full lifecycle management.

Realtime dispatch & tracking

Socket.IO live location, heartbeat monitoring, driver assignment retries, and dispatch heatmaps are implemented in code.

Built for trust

KYC, OTP trip control, SOS, audit trails, and safety compliance are baked into the platform.

Repo feature set

Mobility modules

  • Ride pooling
  • Airport rides
  • Corporate portal
  • Fleet profitability

Driver & earnings

  • Driver trust scoring
  • Referral and earnings analytics
  • Vehicle and ride-type management

Operations & safety

  • Support tickets, disputes, refunds
  • Admin pricing, subscriptions, KYC workflows
  • Monitoring, analytics, compliance

Tech stack

Frontend

Expo / React Native / Web, Google Maps, Stripe mobile SDK, real-time client state, async storage.

Backend

FastAPI; SQLAlchemy/Postgres; Socket.IO; Stripe; Redis optional; Sentry; Prometheus; Stripe webhooks.

Infrastructure

Docker deployment assets, CI/CD, production readiness docs, log/metric hooks, upload storage backends.
